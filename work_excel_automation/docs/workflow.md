# ExcelFlow ワークフロー設計書

安全・確実なExcelファイル編集のための7フェーズワークフローです。

## 7フェーズ詳細

1. **Intake (取り込み)**
   - 入力ファイルを `working` ディレクトリにコピー。
   - UUIDとタイムスタンプから `run_id` を発番し、追跡に用いる。
2. **Discovery (探索)**
   - シート構成、使用範囲、先頭行プレビューをReadOnlyで取得。
   - ヘッダ候補と簡易な列プロファイルを推定する。
3. **Profile Matching (分類)**
   - 探索結果を既知のYAMLプロファイル（Sales, Project等）と照合。
   - 最もスコアの高いプロファイルを特定。未対応の場合は `generic` プロファイルへフォールバック。
4. **Planning (計画)**
   - 人間の指示・LLMの推論から導出された「操作一覧（ops）」と「プロファイルの前提条件」を合成し、`Plan JSON`を確定させる。
5. **Dry Run (ドライラン)**
   - Planの前提条件が現在のファイルと一致しているかチェック。
   - 変更されるシート数、セル数を概算し、閾値（例えば1000セル）を超える場合は安全装置を発動。
6. **Execution (実行)**
   - Planで定義された通りの操作だけを厳密に実行（Plan-then-Act）。
   - 別名保存し、原本は絶対に上書きしない。エラー時は中断。
7. **Verification (検証)**
   - 実行前後のファイルを比較・検証。
   - 想定外のシートが変更されていないかを確実に見る。

## Safe Mode vs Trusted Mode (将来要件)
- **Safe Mode**: 探索・計画・ドライランの結果を人間が確認し、承認してから実行するモード。（PoCではデフォルト想定）
- **Trusted Mode**: プロファイルの一致率が1.0かつドライランの変更セル数が極小（定型操作）の場合、全自動スルー実行するモード。
