# Changelog

All notable changes to this project will be documented in this file.

## [0.1.5.0] - 2026-04-05

### Changed
- **文字起こしアーキテクチャ刷新**: 録音中のGeminiチャンク送信を廃止。録音中はWeb Speech APIのみでリアルタイム表示し、録音停止後に全音声を一括でGemini文字起こし。ハルシネーション（架空の会議内容を生成する問題）を根本解決
- **Geminiプロンプトにハルシネーション防止指示を追加**: 「無音時は空文字を返す、内容を創作しない」を明示。temperature=0.0で創造的出力を抑制
- **長時間録音のffmpegセグメント分割**: 20MB超の音声ファイルはffmpegで10分ごとに自動分割してからGeminiに送信

### Added
- **全音声一括文字起こしAPI**: `POST /api/meetings/{id}/transcribe-full` — 録音停止後に全音声を受け取り、必要に応じてセグメント分割してGemini文字起こし

## [0.1.4.0] - 2026-04-05

### Fixed
- **会議メモ→課題変換が動作しない問題を修正**: 存在しない `POST /api/issues` を呼んでいた3箇所を正しい `POST /api/issues/capture` に修正。レスポンス構造 `{issue: {...}}` への対応も追加
- **IME入力中のEscapeで編集が破棄される問題を修正**: 日本語入力の変換キャンセル時にメモ編集モードが意図せず終了する問題を `isComposing` チェックで解消

### Changed
- **会議ライブメモをNotionライクな複数行入力に改善**: 1行テキスト入力をtextareaに変更。Shift+Enter改行、自動リサイズ、複数行表示対応。録音しながら自由にメモを書ける体験に
- **課題/タスク変換ボタンに重複送信ガードを追加**: 変換処理中はボタンを無効化し、ダブルクリックによる重複登録を防止

## [0.1.3.0] - 2026-04-05

### Added
- **Notionエクスポート**: 会議詳細ページから1クリックでMarkdown形式の会議データをクリップボードにコピー。Notionに貼り付けるとそのまま構造化されたページに
- **ワンクリック課題/タスク変換**: 会議中のライブメモから直接、因果グラフの課題やタスク管理のタスクを起票できるボタンを追加。変換後は「課題 #12 作成済み」のバッジが表示
- **過去決定事項パネル**: 会議詳細ページに同一プロジェクトの過去会議で記録した決定事項を一覧表示。「前回何を決めた？」に即座に回答
- **タスク管理の戻るボタン**: ヘッダー左端にブラウザ履歴ベースの戻るボタン

### Changed
- エクスポートMarkdownの構造改善: AIサマリーを先頭に、決定事項/アクション/リスク/メモをカテゴリ別に分類出力

### Fixed
- **launchd起動パスの修正**: 旧iCloudパスを参照していたlaunchd_start.shを正しいローカルパスに修正。本番サーバーで新しいコード変更が反映されない問題を解消

## [0.1.2.0] - 2026-04-05

### Added
- **タスク依存関係CRUD**: 先行/後続タスクの作成・取得・削除API + 循環依存検知 + TaskDetailPanelにUI追加
- **繰り返しタスク生成**: 設定済みルールに基づき1時間ごとにタスクを自動生成するバックグラウンドスケジューラー + 手動トリガーAPI

### Fixed
- **タスク詳細取得にラベル・マイルストーン情報が欠落**: GET /api/tasks/{id}、サブタスク取得、更新レスポンス等6箇所のSELECTを共通拡張クエリに統一
- **フィールドをnullにクリアできない問題**: category_id/milestone_id/assignee_name等8フィールドのupdate_taskでmodel_fields_set検知に移行、フロントエンドもnull送信に修正
- **子タスク削除が1階層のみ**: 再帰的_delete_task_tree()に置き換え、孫タスク以深も正しくカスケード削除
- **Bulk update時にparent progressが再計算されない**: status変更時に影響parent_idを取得し_recalc_progress()を呼び出し
- **AIチャットで作成されるタスクにproject_name/assignee_nameが設定されない**: 2箇所のINSERTとGeminiプロンプトを拡張
- **議事録抽出タスクにsource_meeting_idが渡らない**: MeetingTaskExtractorからbulkCreateにmeeting_idとsource_typeを伝播
- **filterProjectがサーバーサイドフィルタに含まれない**: fetchTasksの依存配列とAPI送信にfilterProject追加
- **pending remindersのレースコンディション**: 個別UPDATEをIN句バッチ更新に統合
- **Geminiプロンプト内の文字化け**: 「な\\u{FFFD}ればnull」→「なければnull」に修正
- **依存関係dep_typeの入力値未検証**: FS/FF/SS/SFホワイトリストバリデーション追加

## [0.1.1.0] - 2026-04-05

### Fixed
- 会議詳細ページの5つのAPI不整合を修正: 存在しないエンドポイント呼び出し、HTTPメソッド誤り、型定義不一致を解消し、編集・再生成・エンティティ抽出・タスク作成が正常動作するように
- サマリー編集がnotesカラムに誤保存される問題を修正（SessionUpdateにsummaryフィールドを追加）
- AI系エンドポイント（finalize/extract-links/create-tasks）のタイムアウトを15秒→60-120秒に延長し、Gemini応答待ちでの不要なエラーを防止
- モバイルUI改善: ボトムナビゲーション、QuickAddSheet、スワイプ操作、レスポンシブ対応
- 会議録音チャンク送信のレースコンディション修正
- ChromaDB embedding function競合によるチャット停止を修正
- LLM出力のホワイトリスト検証強化（entity_type、chain-risk-scanキー）
- Geminiモデル名を実在するgemini-3-flash-previewに修正

## [0.1.0.0] - 2026-04-04

### Added
- **タスク管理PM業務アップグレード**: 5つのビューモード（今日/ボード/リスト/カレンダー/マップ）、担当者・ラベル・サブタスク・マイルストーン対応
- **AIレポート生成**: タスク進捗から週次/月次ステータスレポートを自動ドラフト
- **議事録→タスク抽出**: 会議の文字起こしからGemini AIがアクションアイテムを検出し、選択的にタスク化
- **テーブルビュー**: スプレッドシート風の一覧表示、ソート・一括操作対応
- **「今日のタスク」ビュー**: 期限超過・今日の期限・進行中タスクをダッシュボード表示
- **URLステート永続化**: ビューモード・フィルタ設定がURL検索パラメータに保持され、リフレッシュ後も復元
- **Evidence Trail**: RAG検索結果に確信度スコアと引用元を表示
- **統一プロジェクトマスター**: 全モジュール横断のプロジェクト管理API
- **会議機能拡張**: アジェンダテンプレート、会議リンクグラフ、ライブメモ、音声再生
- **クロスRAG検索**: 複数ナレッジベース横断検索、カスタム辞書、自動分類
- **AI自動エンティティリンク**: 課題間の関係をAIが自動検出しリンク
- **課題の法的リスク評価 (D2)**: AIが契約・法規制・クレームリスクを評価。ユーザー確認後にノードにバッジ表示。RAGで関連法規を参照
- **エビデンス管理 (D4)**: 課題ごとのエビデンス状態管理。法的リスク中以上の未記録課題をグラフ上でハイライト。推奨エビデンスをルールベースで提案
- **技術コンテキスト解説 (D1)**: RAGで技術基準・仕様書を検索し平易な解説と関連条文を提示。発注者報告用に専門用語を翻訳
- **技術基準クイックリファレンス (D5)**: 課題に関連する技術基準・法規をカード形式で表示
- **因果チェーン横断リスク分析 (D3)**: 因果チェーン全体をAIが横断分析。技術・法的リスク、エビデンスギャップ、推奨アクションを表示
- **Contextual Chunking**: RAGチャンクに文脈ヘッダーを自動付加し検索精度を改善

### Changed
- タスク管理のpage.tsxを11コンポーネントに分割し保守性を向上
- バックエンドタスクAPIを20以上の新エンドポイントで拡張
