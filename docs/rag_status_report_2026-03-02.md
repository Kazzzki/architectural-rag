## 2. ローカル `data/` の状態

ローカルの `data/` ディレクトリ配下には過去に処理した形跡が多数残っており、以下の主要なサブフォルダとその役割に分かれています。（※ディレクトリが初期化されておらず、ファイルが蓄積されています）

- `data/error/`: 処理中にエラーとなった大量のPDFファイルが蓄積されています。
  - 例: `pdf_1771727342_11.pdf`, `test_upload.pdf`, `目で見るアスベスト建材第2版.pdf` など
- `data/input/`: RAGへ登録するための抽出用元データやMarkdownへの変換過程のファイル群があります。
- `data/knowledge_base/`: RAGへ同期された、もしくはアップロードされた「意匠編」「構造編」などの各種PDFやテキストとそのチャンクファイル。以下のようなフォルダ構成を持っています。
  - `04_リサーチ成果物`
  - `1_意匠設計`
  - `2_構造設計`
  - `3_法規チェック`
  - `4_設計マネジメント`
  - `5_コストマネジメント`
- `data/parent_chunks/`: ドキュメントをフラグメント（チャンク）単位に分割した結果のフォルダ群。各PDFの識別子（`54ac99516dcf219a` など）を持つフォルダが多数あります。
- `data/pdfs/`: ダウンロード/保存されたPDFファイル群。
- `data/secrets/`: Drive APIなどで使われる認証用トークン (`credentials.json`, `token.pickle`)。
- SQLite管理ファイル類 (`files.db`, `files.db-shm`, `files.db-wal`) やインデックス情報 (`file_index.json`) も存在しています。
